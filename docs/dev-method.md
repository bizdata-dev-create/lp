# 開発フロー

## 開始手順

1. **ターミナル表示** → `Ctrl+J`
2. **最新コード取得（誰かが変えたかもしれないので最新状態にアップデート）** → `git stash`→`make pull-main`
3. **開発サーバー起動（開発のブラウザを出す）** → `npm run dev`

## 終了手順

**必ず本番環境にアップロード** → `make git-push`

---

# 生成AI指示のコツ

効果的な指示には以下の3点を明確にしてください：

1. **どの部分か** → コードを選択して `Ctrl+L` で生成AIコンテキストに送信
2. **何を変えるか** → 色・文字の太さ・背景・要素配置・縦幅など
3. **どのように変えるか** → 具体例「青 → 濃いめの赤」「中央揃え」「現状の70%程に」

# HTML/CSS の基礎知識

- **`div` タグ** → 配置・レイアウト（構造）を指定
- **`className="..."`** → スタイル（デザイン）を指定
  - 色・配置・枠線など

---

# 編集対象ファイル

基本的に以下のファイル・フォルダのみ編集してください：

- `src/pages/home/page.tsx` → メインページのコンポーネント
- `src/img/` → 画像アップロード用

---

# 編集ワークフロー

1. ページ上の気になるところを見つける
2. そこがsrc/pages/home/page.tsxコードのどこになるか特定する
3. コード部分を選択 → `Ctrl+L` で生成AI に送信
4. チャットで「**何を**」「**どのように変えてほしい**」か明確に指示
5. 'Ctrl+S'でコード保存し、開発用ブラウザを見て、実際にイメージ通りになっているかを確認。なっていなかったら微調整。大幅に悪くなった時はCtrl+Zで戻す
6. ステップ 1～5 を繰り返す

# 初心者向け Web デザイン知識

## よくある質問

### Q: どうして同じコンポーネントでも見た目が違う？
**A:** `className` の指定が異なるため。同じ HTML 構造でも CSS（デザイン指定）が変わると見た目が変わります。

### Q: 色をもっと明るく/暗くしたい場合は？
**A:** Tailwind CSS では色に番号がついています。例：`blue-500` → `blue-400`（明るく）、`blue-600`（暗く）

### Q: 複数の修正をまとめて指示してもいい？
**A:** はい。ただし「**何を・どこを・どのように**」が明確なら、複数指示は効率的です。

### Q: 画像がぼやけているときは？
**A:** 画像ファイルの解像度が低い可能性。高解像度（1920px 以上推奨）の画像を `src/img/` にアップロードしてください。

---

## 開発を捗らせるコツ

1. **定期的に保存＆確認** → コード修正後、ブラウザをリロードして反映を確認
2. **差分を小さくする** → 一度に大量変更せず、細かく区切って指示
3. **色・サイズは統一** → デザイン統一のため、既存の色・余白を参考に
4. **モバイル対応を意識** → `sm:`、`md:`、`lg:` プレフィックスはレスポンシブ指定
5. **エラーメッセージを読む** → ブラウザコンソール（F12）でエラーを確認すると原因特定が早い

# Tailwind CSS よく使うクラス一覧

## 余白（Margin / Padding）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| 上下左右の余白 | `p-4` | パディング 16px |
| 外側の余白 | `m-4` | マージン 16px |
| 上だけ | `mt-4` | マージン上 16px |
| 左右だけ | `mx-4` | マージン左右 16px |
| 横幅いっぱい | `w-full` | 親要素いっぱい |
| 最大幅制限 | `max-w-7xl` | 最大幅 80rem |

## 色（Text / Background）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| 文字色 | `text-blue-600` | 青色の文字 |
| 背景色 | `bg-blue-600` | 青色の背景 |
| 文字グレー | `text-gray-600` | グレーの文字 |
| 背景白 | `bg-white` | 白い背景 |
| 背景グレー | `bg-gray-50` | 薄いグレー背景 |

## 枠線（Border）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| 枠線太さ | `border-2` | 2px の枠線 |
| 枠線太さ | `border-4` | 4px の枠線（太め） |
| 枠線色 | `border-blue-600` | 青色の枠線 |
| 角丸 | `rounded-lg` | 中程度に丸い角 |
| 角丸（大） | `rounded-2xl` | 大きく丸い角 |
| 影 | `shadow-lg` | 大きめの影 |

## テキスト（Font）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| 太字 | `font-bold` | 太い文字 |
| 中程度 | `font-semibold` | やや太い文字 |
| 文字サイズ | `text-xl` | 大きいテキスト |
| 文字サイズ | `text-sm` | 小さいテキスト |
| 中央揃え | `text-center` | テキスト中央揃え |
| 行間 | `leading-relaxed` | 行間を広げる |

## レイアウト（Grid / Flex）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| 横並び | `flex` | フレックスボックス |
| 縦並び | `flex-col` | フレックス縦方向 |
| 2列グリッド | `grid grid-cols-2` | 2列のグリッド |
| PC時2列 | `lg:grid-cols-2` | PC サイズ時のみ 2 列 |
| 中央揃え | `items-center` | 縦方向中央 |
| 中央揃え | `justify-center` | 横方向中央 |
| 空間均等 | `gap-8` | 要素間の間隔 |

## 表示/非表示（Display）

| 用途 | クラス例 | 説明 |
|------|--------|------|
| スマホ非表示 | `hidden md:block` | モバイルで非表示、PC で表示 |
| スマホのみ表示 | `md:hidden` | PC で非表示、モバイルで表示 |
| 不透明度 | `opacity-50` | 50% の透明度 |

## レスポンシブプレフィックス

| プレフィックス | 対応画面幅 | 用途 |
|--------------|----------|------|
| `sm:` | 640px 以上 | タブレット以上 |
| `md:` | 768px 以上 | タブレット以上 |
| `lg:` | 1024px 以上 | パソコン以上 |
| `xl:` | 1280px 以上 | 大きい画面 |

## 使用例

```tsx
{/* 中央に配置した白い背景のカード、グレー文字 */}
<div className="bg-white p-6 rounded-lg shadow-lg text-center">
  <h3 className="text-2xl font-bold text-gray-900 mb-4">タイトル</h3>
  <p className="text-sm text-gray-600 mb-6">説明文</p>
</div>

{/* PC は 2 列、モバイルは 1 列 */}
<div className="grid grid-cols-1 lg:grid-cols-2 gap-8">
  <div>要素 1</div>
  <div>要素 2</div>
</div>

{/* 上下左右の余白、枠線、角丸 */}
<div className="border-2 border-blue-600 rounded-lg p-4 m-4">
  内容
</div>
```
